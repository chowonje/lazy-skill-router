# Router architecture paired A/B — 2026-07-13

## 결론

이 실행은 capability Top-K가 일부 전용 skill 탐색에서 legacy route를 구제할 가능성을 보여줬다. 그러나
한국어와 no-skill 경계가 퇴행했고 최종 ownership 단계가 없어 **behavior promotion은 차단**한다.

Shadow code merge는 기존 hook 선택과 activation을 바꾸지 않는다는 전제 아래 **조건부 검토 가능**하다.
이 판단은 retrieval-first나 ownership bridge 활성화를 승인하지 않는다.

## Gate 판단

### Gate 1 — shadow code merge

**조건부 준비.** Legacy 선택을 바꾸지 않는 독립 measurement lane으로만 검토할 수 있다. 전체 테스트,
hook output equivalence, fail-open, 개인정보 경계, 패키지 검증과 CI가 모두 통과해야 한다.

### Gate 2 — behavior promotion

**실패.** B Recall@3 `73.08%`는 권장 `95%` 미만이고, no-skill 및 한국어 slice가 퇴행했다. Ownership
bridge와 end-to-end activation/outcome evidence도 아직 없다. Top-K labelled conflict는 ranking 품질 지표이지
runtime eligibility 위반이 아니다.
Expected-abstain lexical no-match recall도 A `75.00%`에서 B `21.88%`로 하락해 절대 `95%`와 legacy 비퇴행
기준을 모두 실패한다. 이 지표는 semantic abstention의 대체 증거가 아니라 보수적인 no-skill 중단선이다.

생성된 `PromotionGateV1` 상태도 `blocked`다. 통과한 항목은 개인정보 검사, positive candidate-only CI,
inventory-ineligible hit `0`, operational failure `0`, p95 `20ms` 이하이다. 차단 사유는 다음과 같다.

Privacy leak scan은 최소 비교 길이를 `1`자로 보수화했으며 raw prompt 누출 없이 통과했다. Case ID는 재현과
추적을 위해 report에 의도적으로 유지된다.

- `evidence_artifact_verifier_unavailable`
- `independent_holdout_missing`
- `independent_adjudication_missing`
- `ownership_observation_missing`
- `activation_observation_missing`
- `outcome_observation_missing`
- `candidate_recall_below_minimum`
- `expected_abstain_no_match_recall_below_minimum`
- `expected_abstain_no_match_regressed`

Manifest나 evidence에 `sha256:` 모양의 revision이 있다는 사실만으로는 독립 검증이 성립하지 않는다. 이는
self-attestation일 뿐이다. Evaluator는 이제 명시적 `--artifact-root` 아래 실제 bytes를 검증할 수 있지만, 이
실행에는 독립 artifact가 제공되지 않아 verification은 `unavailable`로 남는다. Byte identity만으로 독립성이나
라벨 품질이 증명되지는 않는다.

이 gate는 `authority: none`, `autoPromote: false`이며 통과하더라도 human review 자격만 부여한다.

다음 실험 전에는 bilingual alias/capability metadata와 retrieval no-match/semantic abstain 경계를 먼저
분리해야 한다. Inventory-ineligible filtering은 index build/load 단계에 이미 존재하므로 gold conflict를
runtime filter로 하드코딩하지 않는다.

## 한계

- Corpus는 synthetic balanced set이며 실제 요청 빈도를 대표하지 않는다.
- 라벨은 단일 작성 흐름이라 독립 adjudication과 inter-annotator agreement가 없다.
- Explicit skill-name 사례는 B에 유리할 수 있다.
- `forbiddenCandidates`는 evaluation label conflict이며 runtime-ineligible 상태를 뜻하지 않는다.
- A는 primary 한 개, B는 Top-3이므로 ranking 지표가 대칭적이지 않다.
- 이 실험은 후보 선택 품질을 비교하며 실제 task outcome이나 host-model ownership 성공률을 측정하지 않는다.
